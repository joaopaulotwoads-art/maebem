2

🔑 Gerar Token do GitHub

O token é como uma "chave especial" que permite ao CNX salvar alterações no seu site. Você só precisa fazer isso uma vez.

🎯 Passo-a-passo simples:

  1. Clique neste link:
    https://github.com/settings/tokens/new?description=CNX+CMS&scopes=repo
    O link já abre com as opções certas marcadas!
  2. Deixe tudo como está e role até o final da página
  3. Clique no botão verde "Generate token"
  4. ⚠️ IMPORTANTE: Copie o token que aparece (começa com ghp_)
  5. Cole o token em um bloco de notas - você vai precisar dele no próximo passo!
⚠️ Cuidado: O GitHub só mostra o token UMA VEZ! Se você não copiar agora, terá que gerar outro. Guarde ele em um lugar seguro.

🤔 Por que preciso de um token?

É uma questão de segurança! O token permite que apenas você (ou o CNX com sua autorização) faça alterações no seu site. É como dar uma chave da casa para alguém de confiança.

3

🚀 Deploy com 1 Clique

Agora é a parte mais fácil! Clique no botão e preencha 4 campos simples. Em 2 minutos seu site estará online!

🚀 Deploy with Vercel

📝 Campos para preencher:

ADMIN_SECRET
Uma senha que você escolhe
Exemplo: minhasenha123 • Será usada para entrar no painel
GITHUB_TOKEN
O token que você copiou no passo anterior
Começa com ghp_ • Cole aqui o que você guardou
GITHUB_OWNER
Seu nome de usuário no GitHub
Exemplo: joaosilva • Clique na sua foto no GitHub para ver
GITHUB_REPO
Nome do projeto (o mesmo que escolheu acima)
Exemplo: meu-site-cnx • Deve ser idêntico ao nome do projeto
🎉 Pronto! Depois que clicar em "Deploy", aguarde uns 2 minutos. Quando aparecer "✅ Success", seu site já está no ar!
4

🎛️ Acessar o Painel Admin

Agora você pode acessar o painel de controle do seu site e começar a personalizá-lo!

🔗 Como acessar:

  1. Acesse: SEU-SITE.vercel.app/admin
  2. Substitua SEU-SITE pelo nome que você deu no deploy
  3. Use o login: admin@admin.com
  4. Use a senha que você colocou em ADMIN_SECRET
🆕 Primeira vez (Configuração Inicial)
  • Preencha: Nome, E-mail, Senha e Confirme
  • Clique em "Criar conta e entrar"
  • Pronto! Você já está logado
🔄 Login normal
  • E-mail: admin@admin.com
  • Senha: valor do ADMIN_SECRET
  • Clique em "Entrar"
🔒 Segurança: Depois do primeiro login, vá em Autores e troque o e-mail e senha para os seus. É muito importante fazer isso!
5

🔒 Trocar E-mail e Senha (Segurança)

⚠️ OBRIGATÓRIO: Faça isso logo após o primeiro acesso por questões de segurança!

🛡️ Passo-a-passo:

  1. No painel, clique em Autores no menu lateral
  2. Clique no usuário "Administrador"
  3. Altere E-mail, Senha e Nome para os seus
  4. Clique em Salvar
✅ A partir daí: Use seu e-mail e senha pessoais para entrar no painel.
6

🔄 Ativar Atualizações Automáticas

⚠️ OBRIGATÓRIO: Sem este passo, o botão "Aplicar agora" não funciona e você não receberá melhorias do template!

⚡ Configurar permissões no GitHub:

  1. Acesse github.com e clique no repositório que foi criado no deploy
  2. Clique em Settings (Configurações) — aba no topo
  3. No menu da esquerda: Actions → General
  4. Role até "Workflow permissions"
  5. Marque ✅ "Read and write permissions"
  6. Marque ✅ "Allow GitHub Actions to create and approve pull requests"
  7. Clique em Save
📖 Guia detalhado: No painel admin, vá em Ajuda → Primeiros passos (passo 3) para ver imagens e mais detalhes.

🔧 Resolução de Problemas

Soluções rápidas para os problemas mais comuns, explicadas em linguagem simples

Soluções por ordem de prioridade:

1. Se você ainda não mudou a senha:
• E-mail: admin@admin.com
• Senha: o valor que você colocou em ADMIN_SECRET no deploy

2. Se o template veio com senha padrão:
• E-mail: admin@admin.com
• Senha: admin123

3. Para trocar a senha:
Depois de entrar, vá em Autores → clique no "Administrador" → altere e-mail e senha.
Solução simples:

1. Vá para vercel.com
2. Entre na sua conta → clique no seu projeto
3. Vá em Settings → Environment Variables
4. Clique em ADMIN_SECRET e mude para uma nova senha
5. Clique em Save e aguarde o redeploy (~1 minuto)
6. Entre com admin@admin.com e a nova senha
Problema: As configurações do GitHub não estão corretas.

Solução passo-a-passo:

1. Vá para vercel.com → seu projeto → Settings → Environment Variables
2. Confira se estas 3 variáveis existem e estão corretas:
• GITHUB_TOKEN (começa com ghp_)
• GITHUB_OWNER (seu usuário do GitHub)
• GITHUB_REPO (nome do repositório)
3. Se alterou algo, faça Redeploy no painel da Vercel
4. Aguarde 2 minutos e tente salvar novamente
Problema: As permissões do GitHub Actions não estão ativadas.

Solução (muito importante):

1. Acesse github.com
2. Clique no repositório que foi criado no deploy
3. Clique em Settings (última aba no topo)
4. No menu da esquerda: Actions → General
5. Role até "Workflow permissions"
6. Marque ✅ "Read and write permissions"
7. Marque ✅ "Allow GitHub Actions to create and approve pull requests"
8. Clique em Save

Agora o botão "Aplicar agora" vai funcionar!
Problema: Alguma variável de ambiente está faltando ou incorreta.

Como descobrir o problema:

1. Vá para vercel.com
2. Clique no seu projeto → Deployments
3. Clique no último deploy (o primeiro da lista)
4. Clique em "View Build Logs"
5. Procure por mensagens de erro em vermelho

Solução: Geralmente é uma variável ausente. Vá em Settings → Environment Variables e confira se todas estão preenchidas.
Duas coisas para verificar:

1. Permissões do GitHub Actions (principal causa):
• Siga os passos do problema "Aplicar agora dá erro" acima

2. Actions ativadas:
• Vá no seu repositório no GitHub
• Clique na aba Actions
• Se aparecer um botão para ativar, clique em "I understand my workflows, go ahead and enable them"

Pronto! As atualizações vão começar a chegar automaticamente.
Não desista! Temos mais recursos para ajudar:

1. Diagnóstico Automático:
No painel do seu site, vá em Configurações → Atualizações → clique em "🔍 Diagnosticar". O sistema vai explicar exatamente qual é o problema e como resolver.

2. Central de Ajuda:
No painel, clique em Ajuda no menu lateral para ver mais soluções.

3. GitHub:
Se ainda tiver problemas, abra uma issue no GitHub do projeto descrevendo o problema. A comunidade vai ajudar!
💡 Lembre-se: 95% dos problemas são resolvidos verificando as configurações no Vercel (Environment Variables) e as permissões no GitHub (Actions). Se alguma coisa não funciona, comece por aí!
